Kubernetes 基礎

このチュートリアルは Kubernetes クラスタ・オーケストレーション・システムの基本を一通りこなします。各要素には主な Kubernetes 機能と概要に関する背景情報や、双方向のオンライン・チュートリアルを含みます。双方向チュートリアルでは、１つのクラスタを管理し、自分自身で コンテナ化したアプリケーション を管理します。

双方向チュートリアルで学べるのは：

  • コンテナ化したアプリケーションをクラスタに展開（デプロイ）
  • デプロイメントのスケール
  • コンテナ化したアプリケーションを新しいソフトウェアのバージョンに更新
  • コンテナ化したアプリケーションのデバッグ

チュートリアルでは Katacoda を使ってウェブブラウザ上に仮想ターミナルを実行します。これは Minikube を実行するものであり、Kubernetes の小規模ローカル環境としてどこでも実行できるものです。使うにはソフトウェアのインストールや設定が不要ですので、それぞれの双方向チュートリアルでは自分のブラウザ以外に使うものはありません。


Kubernetes は何ができるのでしょうか？

最近のウェブサービスでは、利用者がアプリケーションを毎日24時間利用できるのを期待しています。そして、開発者はこのアプリケーションの新しいバージョンを１日に何度も展開したいでしょう。停止時間（ダウンタイム）なく簡単かつ素早いアプリケーションのリリースと更新を行えるよう、コンテナ化はソフトウェアのパッケージ化は、これらの目標を達成するために役立ちます。Kubernetes はどこでも必要な時にこれらコンテナ化したアプリケーションを確実に動かすために役立つものであり、これら処理に必要なリソースとツールを見つけるためにも役立つでしょう。Kubernetes は本番に対応しているオープンソースのプラットフォームであり、設計に基づいているのは、Google が蓄積したコンテナ・オーケストレーションの経験と、コミュニティによる最善の組み合わせのアイディアを連携です。
