Kubernetes とは、コンテナ化したアプリケーションのデプロイ、スケール、管理を自動化するオープンソースのシステムです。

コンテナのグループ化とは、管理とディスカバリが簡単になるように、アプリケーションを論理的な集まりに整理します。Kubernetes の基礎を積み上げるのは Google における 15 年もの本番環境向け作業における動作経験であり、 これらとコミュニティからのアイディア及び経験の最善の組み合わせが一体となったものです。

世界規模のスケール

Google が毎週 10 億ものコンテナを動かすのと同じ方式で設計しているため、Kubernetes は皆さんの運用チームを増員しなくてもスケールできます。

成長は未だに継続

ローカルにおけるテストやグローバルなエンタープライズにおいても、Kubernetes の複雑さが増す心配は不要であり、アプリケーションを一貫して簡単に供給し、皆さんと共に柔軟に成長します。

あらゆる場所で実行

Kubernetes はオープンソースであり、オンプレミス、ハイブリッド、あるいはパブリッククラウド基盤の利点を自由に得られるため、ワークロードを効率的かつ心配する必要なくどこでも移動可能にします。

Kubernetes 機能

自動的に詰め込む(binpacking)

障害が起こればコンテナを再起動し、ノードが壊れればコンテナの再配置と再スケジュールをし、定義したヘルスチェックの応答がなければコンテナを停止し、クライアントに提供する準備が整わなければ広報(advertise)しません。

自己修復 (Self-healing)

障害が起こればコンテナを再起動し、ノードが壊れればコンテナの再配置と再スケジュールをし、定義したヘルスチェックの応答がなければコンテナを停止し、クライアントに提供する準備が整わなければ広報(advertise)しません。

水平スケール

簡単なコマンドか UI あるいは CPU 使用率に基づき、アプリケーションを自動的にスケールアップ・ダウンする。

サービス・ディスカバリと負荷分散

サービス・ディスカバリ機構になれていなくても、アプリケーションに手を加えることなく利用できる。Kubernetes はコンテナに対して IP アドレスとコンテナ群に対する１つの DNS 名をあたえ、相互に負荷分散できるようにする。

自動的なロールアウトとロールバック

Kubernetes はアプリケーションの正常性を確保したまま、全てのインスタンスを停止せず、アプリケーションや設定に対する変更を徐々に適用する。もし何か問題が起これば、Kubernetes は変更に対してロールバックする。

シークレット と 設定 の管理

シークレットとアプリケーション設定の展開と更新には、イメージの再構築やスタック設定内でシークレットを露出しない。

ストレージ・オーケストレーション

ローカル・ストレージや GCP や AWSなどのパブリック・クラウド・プロバイダ、あるいは NFS や iSCSI、Gluster、Ceph、Cinder、Flocker のような、選択に応じたストレージ・システムを自動的にマウントする。

バッチ実行

サービスだけでなく、Kubernetes はバッチや CI ワークロードなども管理でき、必要があれば障害時にコンテナを置き換えられる。
